About CSS3

CSS3 is the newer version of CSS used for styling web pages more effectively. CSS3 has introduced a lot of enhanced features. CSS3 is divided into modules which makes styling of the web pages simpler and easier.

Developers choose CSS3 to style web pages without using any additional tools. It helps in creating attractive web pages and makes the web layout very simple. CSS3 can help in creating dynamic web pages without using JavaScript. It also helps in creating responsive web pages.

We can use the style attribute to specify the styling. This is referred to as inline styling. We can style only one element at a time using Inline styling

Internal Styling

We can specify the style rules separately within the style tag in an HTML web page. This is referred to as internal/embedded styling as we embed the styling along with HTML

Internal styling affects only the HTML elements of the webpage in which the internal style rules are written

    <!DOCTYPE html>
    <html>

Syntax:

    <style type="text/css">
        @import url("http://www.xyz.com/style.css")
   </style>

Selectors

why Selectors ?

Selectors are used for targetting the HTML elements and style them appropriately.

What are Selectors ?

There are different types of selectors available in CSS.

Element Selectors

Element selectors are used for styling elements

We can style the elements by selecting them using element selector as shown below

    body {
        background-color: lightgrey;
      }

In the above webpage, the body is applied with a light grey background-color using element selectors

Id Selector

Id selectors are used for styling an element specifically

We can use the id selector as shown below

    #pid {
        background-color: yellow;
        color: green;
      }

We can add an id attribute to the HTML element as shown below

    <p id="pid">Paragraph1

The first paragraph is styled with green color and yellow background-color using the id selector

Class Selectors

Class selectors are used for providing the same styling for multiple elements.

Class selectors can be used as shown below

    .red {
        color: red;
    }

We apply a class to the HTML using the class attribute

    <p class="red">Paragraph1</p>
    <p class="red">Paragraph2</p>

All the paragraphs are styled in red color using class selectors.

Contextual Selectors

Contextual selectors are used when we want selectors to match elements that appear in a context. Context is referred to as a parent/child relationship between HTML elements. We separate the parent and child elements using space while writing the style rules using contextual selectors.

We can use contextual selectors as shown below

    <div>
        <p class="red">Paragraph1</p>
        <p class="red">Paragraph2</p>
    </div>
    

    div p {
        background-color: yellow;
        color: green;
      }

The first two paragraphs are present within the div element When styled using div p contextual selectors, it styles only the first 2 paragraphs and not the next 2 paragraphs

Grouping of selectors

When we have the same styling for multiple elements we can group the elements and style together. Grouping of selectors helps in reducing the repetitive styling statements. The multiple HTML elements that we group to provide a common styling should be separated using a comma.

We can group the selectors as shown below

    h1,h2,h3 {
        color: maroon;
      }

The text CSS, Selectors, and Grouping are rendered using h1, h2, and h3 elements. Since all the headings need the same styling, we can group the selectors.

Inheritance

The child elements can inherit the style properties from the parent element, but some of the stylings cannot be inherited by child elements automatically.

Some of the CSS style properties have “inherit” value associated with them. For example, Font and Text properties are inherited but the border and margin properties are not inherited.

If the “inherit” value is associated with the property then styling is also inherited. Otherwise, styling is not inherited.

If we want the styling which is not inherited by default to be inherited then we have to specify inherit to the style attribute explicitly.

Example: 

    border: inherit

Both the first and second paragraphs are present within a div tag, and the div tag is applied with color and background-color, but the paragraphs are not applied with any styling. But the paragraph tags inherit the styling from their parent element.

Cascading Order

Why Cascading order ?

Consider a company that has multiple roles. An employee needs to complete some tasks as part of his/her job. Assume his /her immediate manager gives a task to an employee. When the employee is working on the task, say the team lead provides another high-priority task, and then suddenly the Project manager assigns another task to the employee. The employee would prioritize the work in the following order

  1. Project manager
  2. Team Lead
  3. Manager
  4. Employee

Similar to the above scenario, CSS also prioritizes when there are multiple styling and applies one styling.

What is Cascading Order ?

CSS stands for Cascading style sheets, the first-word cascade is important to understand. When there is a conflict between multiple style rules, which rule to apply is decided using cascade and specificity concepts.

If styling is mentioned using multiple ways ultimately they cascade to one style.

Priority order/ Cascading order

  1. Inline
  2. Internal and/or impor
  3. External
  4. Browser default

Selector’s precedence

If there are multiple styles specified for the same HTML element then the priority is as below.

  1. id
  2. class
  3. element

Thumb Rule: If two rules have the same weight, the latter wins.

For example, the internal style sheets and imported style sheets carry the same weight, but since imported styles must always appear at the start of the style sheet, the latter will win i.e., internal style sheets.

Specificity

When there are multiple styles specified using multiple selectors in a style rule, then the priority is to be decided based on the specificity

A Specificity is a 3-digit number. if the specificity of a style rule is 123 then,

1 refers to the number of id selectors

2 refers to the number of class selectors

3 refers to the number of elements

The style rule with the highest specificity always wins.

    <section class="sclass" id="sid"> 
        <p class="pclass" id="pid">Cascading order</p> 
    </section> 
    

    #sid #pid {
        color: blue;
      }
      section.sclass p#pid.pclass {
        color: green;
      }

There are 2 styling statements and both are styling the paragraph element present within the section element

The specificity of the first statement is 200 (2 id selectors, 0 class selectors, and 0 element selectors)

The specificity of the second statement is 122 (1 id selector, 2 class selectors, and 2 element selectors)

The highest specificity (200) wins so the first styling statement will be prioritized.

!important Keyword

In case if any style property is important, and should not be overwritten by any, other selectors, we can specify the !important keyword against the property. 

    color: limegreen !important; 

This property will not be overwritten, even if the specificity of the selector is the largest.

Overview of Cascading Order

Selectors priority:

    <body> 
        <section class="sclass" id="sid">

      p {
        color: green;
      }

Line 2: Targeting the paragraph elements directly will win over the inheritance. Hence this styling statement wins and the text color of the paragraph will be green.

Hence paragraph will be rendered in green color

!important Keyword:

    #pid {
        color: red !important;
      }

The paragraph with id "pid" will be rendered with red color regardless of any other rule applied.

The style rule is applied with !important. The !important keyword should be specified after the style rule preceded with "!" mark

Specificity:

<section class="sclass" id="sid"> <p class="pclass" id="pid">Cascading order</p> </section> 
    p#pid {
        color: green;
      }
      #sid p {
        color: red;
      }

Both the above styling statements carry the same weight because both have 1 id and 1 tag but since the red color appears latter it wins

Hence paragraph will be rendered in red color

    #sid #pid {
        color: blue;
      }
      section.sclass p#pid.pclass {
        color: green;
      }

Line 1: The specificity of this statement is 200 (2 ids)

Line 2: The specificity of this statement is 122 (1 id, 2 class, and 2 tag names).

The highest specificity (200) wins so the text color of the paragraph will be blue.

Typography

Why Typography ?

Typography refers to the art of arranging type.

Typography plays a very important role in advertising and sales.

The right choice of scale of the text, color, and most importantly the font not only makes the readers curious to read the content completely sometimes also elicit some emotions.

What is Typography ?

Typography refers to the technique of presenting the text of a website in a legitimate, readable, and appealing way Elements of good typography included

  • Color: Background and text-color
  • Spacing: letter-spacing,line-height
  • Font: font-style and font-size
  • Alignment: text-align

The choice of background color and text color and font makes the web page attractive. However, the proper colors should be chosen so that the text is readable. The wrong choice of color makes the text unreadable.

The font-family chosen adds more aesthetic value to the web page. The font-size should be properly decided. The headings should be displayed with bigger font and paragraph text with a lesser font. Aligning the text appropriately also plays a major role in good typography.

Color Models

In CSS3, color models have been introduced by using which we can customize the default color of HTML element with opacity property. Let us learn about opacity property.

We can use the opacity property to change the opacity or transparency level of the HTML elements. Using this property the opacity of the entire element changes.

    div
    {
       background-color:red;
       opacity:0.5;
    }
  • The opacity value ranges between 0 and 1 where 0 indicates fully transparent and 1 indicated fully opaque.
  • The default value of opacity is 1.If the parent element has this property then all its child elements will have the same opacity applied.

Why Box Model ?

CSS box model can be related to the above example.

Each photo frame can be considered as an HTML element. The photo present in each frame can be considered as the HTML content. The spacing from the photo to the frame is equivalent to padding. The frame itself is the border, The distance between the inner boundary to the outer boundary (the width of the brown portion in the above image shown ) is equivalent to the border width. The spacing between one frame to another frame or the wall is the margin.

What is Box Model ?

The CSS box model is a container, which wraps around every HTML element. It consists of margins, borders, padding along with the actual content.

The space from the content to the border is padding. The green box around the element is the border and the thickness of the border is the width of the border. The spacing from the HTML element to the neighboring elements is the margin. The width and height of an element include padding, border, and margin.

The padding, margin, and border are the CSS properties, which helps in providing proper spacing to the HTML element so that the webpage is readable by the end-users.

CSS short hand proerties

CSS has many shorthand properties. The shorthand properties in CSS can take from 1 to 4 values. The order of these values are top, right, bottom and left

If 1 value is specified then the same value is considered for all other 3 values.

If 2 values are specified then the first value is considered as top and bottom values and the second value is considered as left and right values.

If 3 values are specified then the first value is considered as the top value and the second value is considered as left and right values and the third value is considered as bottom value.

Transformations

Why Transformations ?

You might have observed an HTML element getting transformed such as the element gets rotated or scaled upon hover of the mouse.

These transformations were generally achieved through JavaScript or JavaScript frameworks.

CSS3 has introduced the transformations using which we can transform our HTML elements using the transform property of CSS3.

Transformations are applied to create dynamic web pages.

CSS3 enables us to move, tilt, scale any element of our web page with ease with the concept of transforms.

We can rotate and pop out a particular image in an image gallery on the hover of the mouse.

CSS3 supports the following transformations

  • Translate
  • Scale
  • Rotate
  • Skew

Translate

CSS3 translate() function is used to move an element from one position to another position.

An element can be moved to the right, left, top, bottom.

Syntax: translate (x,y) -> moves the element to right by x-value, and y-value down from its current position.

An element can be moved in only one direction i.e. horizontal/vertical using translateX and translateY functions.

Example:

  1. translate(20px, 30px) -> moves the element to right by 20px and 30px down from the current position.
  2. translate(-40px, 20px) -> moves the element to left by 40px and 20px down from the current position.
  3. translate(-30px, -25px) -> moves the element to left by 30px and 25px up from the current position.

Transitions

Why Transitions ?

If we have a requirement to show the gradual changes in the CSS properties of an element during an interval of time.

For example, you might want to show an element changing its size in an interval of 2 seconds upon hover of the mouse. We can use the CSS3 transition property.

What are Transitions ?

Transitions are used for moving the element from one state to another state by transiting the CSS properties of an element over a period of time.

We can mention the initial state and final state with transitions and it happens when some event occurs.

Observe how the movement of the rocket can be transited during a time interval on a webpage.

Transitions are gradual changes in the properties of an element when it changes from one style to another.

Using CSS3 transition effects can be added to elements without the need for any scripting or additional software like Flash.

To add a transition, two things must be specified:

  • The CSS property to which the effect should be added.
  • The duration of the effect.
    transition: height 3s;

height is the CSS property to be transited.3s is the duration for the animation

Transition is a short-hand property to specify the properties mentioned below

Transition Timing Function

The transition timing function is used to specify the pace/speed of transition. This can be done in 2 ways.

Specifying the pre-defined functions.

Defining a custom function by specifying four co-ordinates to define a cubic bezier curve.

The Cubic Bezier curve is defined by 4 points P0, P1, P2, and P3.

P0 and P3 are the start and end of the curve hence they have values 0 and 1 by default. We specify the x and y coordinate values for P1 and P2 to the cubic bezier curve function.

    cubic-bezier(x1,y1,x2,y2)

In the above syntax, x1,y1,x2, and y2 are the x and y-coordinates of points P1 and P2.

Steps Transition Function

steps() timing function enables us to break the transition into steps.

For example: If I want to transit a square that moves 400px to right on hover in 4 seconds then the square would move 100 pixels to right for every second.

This function accepts 2 arguments.

steps(int, start|end)

The first parameter specifies the number of intervals to which the transition should be broken. This should be a positive number greater than 0.

If the second parameter value is 'start' then the transition will happen at the beginning of each step.

If the second parameter value is 'end' then the transition will happen at the end of each step.

There are 2 pre-defined variants of the steps function

  • There are 2 pre-defined variants of the steps function
  • step-end which is equivalent to steps(1, end)
    div
    {
        transition: 4s steps(4);
    }
    div:hover
    {
        left: 400px;
    }

Line 3: The transition will be broken into 4 intervals.
